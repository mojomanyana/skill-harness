# skill-check ŌåÆ skill-harness: rebrand + publish ŌĆö Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the project `skill-check` ŌåÆ `skill-harness` (packages, CLI command, repo, docs, committed bundle) and make it publish-ready (scoped `@skill-harness/{core,adapters,cli}` + a thin unscoped `skill-harness` meta-package), so users install the CLI from npm and consume the Action via a `v1` tag.

The three outward-facing/irreversible actions ŌĆö `npm publish` (user-run), `gh repo rename` + `git tag v1` (agent-run, user-gated) ŌĆö are terminal steps, not tasks.

**Tech Stack:** TypeScript ESM (relative imports end in `.js`), node sync `fs`, `js-yaml`, vitest, esbuild. No new runtime deps.
