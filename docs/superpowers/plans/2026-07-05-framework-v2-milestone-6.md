# skill-check v2 ‚Äî Milestone 6: pi-extension package ‚Äî Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship skill-check as a pi extension ‚Äî a `/skill-check` slash command (run/judge/review) plus a `skill_check_run` tool the model can call ‚Äî installable via `pi install git:github.com/mojomanyana/skill-check`, importing `@skill-check/core` in-process and streaming verdicts into the pi session.

**Architecture:** New `packages/pi-extension` whose `src/index.ts` default-exports `(pi: ExtensionAPI) => {}`, registering the command + tool. It calls `runSkillModel`/`regradeRun`/`serveReview` in-process, routing core's `onProgress` log callback to the pi session UI. Built with esbuild into a single self-contained `dist/index.js` (core/adapters inlined; `@earendil-works/*`+`typebox` external), committed to the repo so `pi install git:` needs no build step.

**Tech Stack:** TypeScript ESM (relative imports end in `.js`), node sync `fs`, `js-yaml`, vitest, **esbuild** (new build-time devDependency), pi's extension API (`@earendil-works/pi-coding-agent`, `typebox` ‚Äî peer-provided by pi).

**Spec:** `docs/superpowers/specs/2026-07-05-skill-check-m6-pi-extension-design.md`. Master: `docs/superpowers/specs/2026-07-03-skill-check-framework-design.md` (migration path step 6). pi extension API reference (on disk): `~/.nvm/.../@earendil-works/pi-coding-agent/docs/extensions.md` + `packages.md`.

**Repo / branch:** `~/prepos/skill-check`, branch `framework-v2-m6` (spec committed `24607a5`). `main` has M1‚ÄìM5b merged. Baseline: 196 tests.

## Global Constraints

- `npm run build && npx vitest run` green at every commit (repo root). Tests run against package `src` via the existing vitest workspace alias.
- ESM: every relative import ends in `.js`. Sync `node:fs`.
- **One new dependency is allowed this milestone:** `esbuild` as a **devDependency** (build-time bundler; already present transitively via vitest, so no install risk). No other new runtime deps. Declare as `peerDependencies: "*"` **only the pi packages actually imported** ‚Äî `@earendil-works/pi-coding-agent` and `typebox` ‚Äî and do **not** bundle them (pi provides them). (Do not declare `@earendil-works/pi-ai`/`pi-tui` unless a file imports them.)
- The extension bundle `packages/pi-extension/dist/index.js` is a **committed build artifact** (checked into git) ‚Äî regenerated by `npm run build:ext` and committed whenever extension source changes.
- Root `package.json` gains `"keywords": ["pi-package"]` and `"pi": { "extensions": ["packages/pi-extension/dist/index.js"], "skills": ["./SKILL.md"] }`. Adding a `pi` manifest **replaces** pi's convention auto-discovery, so the `skills` entry is REQUIRED to preserve the existing `./SKILL.md` front door.
- `--no-extensions` is added ONLY to the `pi`-binary child invocations (subject run + non-`claude-code` judge), NOT the `claude` binary branch. It must not break `--skill` (green mode).
- `Scorecard` shape (produced by the runner, consumed by command + tool + tests): `{ skill: string; model: string; grade: { pct: number; letter: string; ship: boolean }; scenarios: { id: string; verdict: "PASS"|"FAIL"|"ERROR"; suspect: boolean }[]; failedTranscripts: string[] }`.
- **Scope note / deliberate deviation:** the design says "M6 only adds the extension package + the one `--no-extensions` adapter change." Task 3 additionally performs a *behavior-preserving* extraction of `regradeRun` from `cmdGrade` (mirroring M5b's `judgeOneRep`) so the extension's `judge` reuses one source of truth instead of duplicating the loop; and Task 2 adds a subpath export + an `assetsDir` option to `serveReview`. Both are minimal, test-locked, and in service of DRY. Flagged for confirmation at execution handoff.

```ts
import { existsSync } from "node:fs";
import { dirname, join, resolve } from "node:path";
import { loadSpec, runSkillModel, effectiveVerdicts, findTranscriptFiles, type HarnessAdapter } from "@skill-check/core";
import { getAdapter } from "@skill-check/adapters";

export interface Scorecard {
  skill: string; model: string;
  grade: { pct: number; letter: string; ship: boolean };
  scenarios: { id: string; verdict: "PASS" | "FAIL" | "ERROR"; suspect: boolean }[];
  failedTranscripts: string[];
}

/** Explicit dir wins; else walk `cwd` upward until a `tests/specification.yaml` is found. */
export function resolveSkillDir(cwd: string, arg?: string): string {
  const start = arg ? resolve(cwd, arg) : cwd;
  let dir = start;
  for (;;) {
    if (existsSync(join(dir, "tests", "specification.yaml"))) return dir;
    const parent = dirname(dir);
    if (parent === dir) break;
    dir = parent;
  }
  throw new Error(`no tests/specification.yaml found from ${start} upward`);
}

// Keep in sync with cli.ts DEFAULT_MODEL / DEFAULT_JUDGE (cli.ts:16-17).
const DEFAULT_MODEL = "fireworks:accounts/fireworks/models/deepseek-v4-pro";
const DEFAULT_JUDGE = { provider: "anthropic", model: "claude-opus-4-8" };
// Prefer core's validated parseModelRef if exported (adapters/types.ts); this local
// fallback must trim + reject empty provider/model to match it.
function parseModelRef(token: string) {
  const i = token.indexOf(":");
  if (i <= 0 || i === token.length - 1) throw new Error(`invalid model token: ${token} (want provider:model)`);
  return { provider: token.slice(0, i).trim(), model: token.slice(i + 1).trim() };
}

export async function runViaExtension(opts: {
  skillDir: string; model?: string; reps?: number; mode?: "red" | "green" | "force";
  adapter?: HarnessAdapter; judge?: string; now?: () => string; timestamp: string;
  log: (msg: string) => void;
}): Promise<Scorecard> {
  const specPath = join(opts.skillDir, "tests", "specification.yaml");
  const spec = loadSpec(specPath);
  const modelToken = opts.model ?? DEFAULT_MODEL;
  const model = parseModelRef(modelToken);
  const judge = opts.judge ? parseModelRef(opts.judge) : DEFAULT_JUDGE;
  const adapter = opts.adapter ?? getAdapter("pi");

    timestamp: opts.timestamp, now: opts.now, reps: opts.reps, onProgress: opts.log,
  });
  const g = summary.results.effective_grade;
  const verdicts = effectiveVerdicts(summary.results.scenarios);
  const failedTranscripts = verdicts
    .filter((v) => v.verdict !== "PASS")

      .map((f) => join(summary.runDir, f)));

    skill: summary.results.skill, model: summary.results.model,
    grade: { pct: g.pct, letter: g.letter, ship: g.ship },
    scenarios: verdicts.map((v) => ({ id: v.id, verdict: v.verdict, suspect: v.suspect })),

  };
}

import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";
import { loadSpec, regradeRun, type HarnessAdapter } from "@skill-check/core";
import { getAdapter } from "@skill-check/adapters";
import { serveReview, type ServeHandle } from "@skill-check/cli/serve";
import { join } from "node:path";
import { resolveSkillDir, runViaExtension } from "./runner.js";

const USAGE = "usage: /skill-check run [skill] [--model p:m] [--reps N] [--mode red|green|force] | judge [run-dir] | review [skill]";

// minimal arg tokenizer: subcommand + positional + --flag value pairs
function parse(args: string) { /* split on whitespace; first token = sub; collect --k v into flags; rest positional */ }

// Emit a line to the session UI when available, else stdout (print/json/-p mode has ctx.hasUI === false).
function say(ctx: any, msg: string, level: "info" | "warning" | "error" = "info") {
  if (ctx.hasUI) ctx.ui.notify(msg, level); else console.log(msg);
}

export async function handleSkillCheck(argstr: string, ctx: any, opts?: { adapter?: HarnessAdapter; assetsDir?: string }): Promise<ServeHandle | void> {
  const { sub, positional, flags } = parse(argstr);
  const adapter = opts?.adapter;
  const nowIso = () => new Date().toISOString();
  if (sub === "run") {
    const skillDir = resolveSkillDir(ctx.cwd, positional[0]);
    const card = await runViaExtension({
      skillDir, model: flags.model, reps: flags.reps ? Number(flags.reps) : undefined,
      mode: flags.mode as any, adapter, judge: flags.judge, timestamp: nowIso(),
      log: (m) => { if (ctx.hasUI) ctx.ui.setStatus?.("skill-check", m); }, // live footer only in TUI
    });
    say(ctx, `${card.skill} ${card.grade.letter} (${card.grade.pct}%) ${card.grade.ship ? "SHIP" : "NOT READY"}`, card.grade.ship ? "info" : "warning");
    for (const s of card.scenarios) say(ctx, `  ${s.id}: ${s.suspect ? "?" : s.verdict}`);
    if (card.failedTranscripts.length) say(ctx, `failed transcripts:\n${card.failedTranscripts.join("\n")}`);
    return;
  }
  if (sub === "judge") {
    const runDir = positional[0] ?? ctx.cwd;
    // derive the spec from the RUN DIR's own skill (results are at <skillDir>/tests/results/<tag>/<ts>/),
    // mirroring cmdGrade (cli.ts:183) ‚Äî NOT from cwd, which could be a different skill.
    const testsDir = dirname(dirname(dirname(runDir))); // <skillDir>/tests
    const spec = loadSpec(join(testsDir, "specification.yaml"));
    const results = await regradeRun({ runDir, spec, adapter: adapter ?? getAdapter("pi"),
      judge: { provider: "anthropic", model: "claude-opus-4-8" }, specDir: testsDir, now: nowIso });
    say(ctx, `re-judged ${runDir}: ${results.effective_grade.letter} (${results.effective_grade.pct}%)`);
    return;
  }
  if (sub === "review") {
    const skillDir = resolveSkillDir(ctx.cwd, positional[0]);
    const spec = loadSpec(join(skillDir, "tests", "specification.yaml"));
    const handle = await serveReview({ skillDir, skillName: spec.skill, port: 0, open: false, adapter,
      assetsDir: opts?.assetsDir }); // threaded from index.ts via the closure, never off ctx
    say(ctx, `review server: http://127.0.0.1:${handle.port}/`);
    return handle; // index.ts registers a session_shutdown to close it
  }
  say(ctx, USAGE);
}

let reviewHandle: ServeHandle | null = null;
/** Close any running review server (called from index.ts's session_shutdown). */
export function closeReview(): void { reviewHandle?.close(); reviewHandle = null; }

export function registerCommand(pi: ExtensionAPI, assetsDir?: string): void {
  pi.registerCommand("skill-check", {
    description: "Run, judge, or review a skill's scenarios",
    handler: async (args, ctx) => {
      const h = await handleSkillCheck(args, ctx, { assetsDir });
      if (h) { reviewHandle?.close(); reviewHandle = h; } // keep the latest review server for shutdown cleanup
    },
  });
}
```

import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";
import { Type } from "typebox";
import { resolveSkillDir, runViaExtension, type Scorecard } from "./runner.js";

export const skillCheckRunTool = {
  name: "skill_check_run",
  label: "Run skill-check",
  description: "Run a skill's scenarios and return the scorecard (grade, per-scenario verdicts, failed transcripts). Use after editing a skill to validate it.",
  promptGuidelines: ["Use skill_check_run after editing a skill to validate it against its scenarios."],
  parameters: Type.Object({
    skill: Type.Optional(Type.String({ description: "skill dir/name; defaults to the current project" })),
    model: Type.Optional(Type.String({ description: "provider:model token under test" })),
    reps: Type.Optional(Type.Number({ description: "run each scenario N times" })),
    mode: Type.Optional(Type.String({ description: "red | green | force" })),
  }),
  async execute(_id: string, params: any, _signal: AbortSignal, onUpdate: any, ctx: any) {
    const skillDir = resolveSkillDir(ctx.cwd, params.skill);
    const card: Scorecard = await runViaExtension({
      skillDir, model: params.model, reps: params.reps, mode: params.mode,
      adapter: ctx.__adapter, timestamp: new Date().toISOString(),
      log: (m) => onUpdate?.({ content: [{ type: "text", text: m }] }),
    });
    const summary = `${card.skill} ${card.grade.letter} (${card.grade.pct}%) ‚Äî ${card.grade.ship ? "SHIP" : "NOT READY"}\n`
      + card.scenarios.map((s) => `  ${s.id}: ${s.suspect ? "? (suspect)" : s.verdict}`).join("\n")
      + (card.failedTranscripts.length ? `\nfailed transcripts:\n${card.failedTranscripts.join("\n")}` : "");
    return { content: [{ type: "text", text: summary }], details: card };
  },
};

export function registerTool(pi: ExtensionAPI): void { pi.registerTool(skillCheckRunTool as any); }
